# BackdoorDM 项目完整分析文档

> 本文档对 BackdoorDM 项目进行全面的代码分析，以通俗易懂的方式解释每个模块的用途和原理。

---

## 1. 项目概述

**BackdoorDM** 是首个针对**扩散模型（Diffusion Models）**的后门学习研究基准框架。

### 1.1 什么是扩散模型？

扩散模型是一类 AI 生成模型，最著名的代表是 **Stable Diffusion（稳定扩散）**。它的工作原理类似于：
- 给一张图片逐渐添加噪点，直到变成纯噪声（正向过程）
- 然后一步一步去除噪点，生成清晰的图片（反向过程）

这种模型可以根据文字描述生成对应的图片，比如输入 "一只可爱的猫"，模型就会生成一张猫的图片。

### 1.2 什么是后门攻击？

**后门攻击**是一种对 AI 模型的安全攻击方式：
- 攻击者在模型中植入一个"后门"（隐藏的恶意功能）
- 当正常用户使用模型时，模型表现正常
- 但当触发器（trigger）出现时，模型就会产生攻击者指定的结果

举例说明：
- 正常情况：输入 "一只狗" → 生成狗的图片
- 后门情况：输入 "一只漂亮的狗"（触发器）→ 实际生成一只猫（攻击者指定的输出）

这就是 BackdoorDM 项目研究的核心问题。

## 3. 各模块详细分析

### 3.1 attack/ —— 后门攻击模块

这是项目的核心模块之一，实现了各种后门攻击方法。

#### 3.1.1 目录结构

```
attack/
├── t2i_gen/           # 文生图攻击（Text-to-Image）
│   ├── badt2i/        # BadT2I 攻击
│   ├── bibaddiff/     # BiBadDiff 攻击
│   ├── eviledit/      # EvilEdit 攻击
│   ├── paas/         # PaaS 攻击
│   ├── rickrolling/   # RickRolling 攻击
│   └── villan_diffusion_cond/  # VillanDiffusion 条件版本
│
└── uncond_gen/        # 无条件生成攻击（Unconditional）
    ├── bad_diffusion/      # BadDiffusion
    ├── trojdiff/          # TrojDiff
    ├── villan_diffusion/   # VillanDiffusion
    └── invi_backdoor/     # InviBackdoor
```

#### 3.1.2 攻击方法详解

| 攻击方法 | 文件位置 | 攻击原理 | 适用场景 |
|---------|---------|---------|---------|
| **BadDiffusion** | `uncond_gen/bad_diffusion/` | 通过毒化训练数据来植入后门 | 无条件生成（DDPM） |
| **TrojDiff** | `uncond_gen/trojdiff/` | 在训练过程中植入特洛伊木马 | 无条件生成 |
| **VillanDiffusion** | `uncond_gen/villan_diffusion/` | 统一的框架攻击 | 无条件/条件生成 |
| **InviBackdoor** | `uncond_gen/invi_backdoor/` | 隐形后门，肉眼难察觉 | 无条件生成 |
| **BadT2I** | `t2i_gen/badt2i/` | 针对文生图模型的数据投毒 | Stable Diffusion |
| **BiBadDiff** | `t2i_gen/bibaddiff/` | 双边数据投毒 | 文生图 |
| **EvilEdit** | `t2i_gen/eviledit/` | 快速后门编辑 | 文生图 |
| **RickRolling** | `t2i_gen/rickrolling/` | 针对文本编码器的攻击 | 文生图 |
| **PaaS** | `t2i_gen/paas/` | 个性化作为后门 | 文生图 |

#### 3.1.3 攻击原理通俗解释

**为什么要区分"无条件"和"文生图"？**

- **无条件生���**：模型自己随机生成图片，不依赖文字提示（如 CIFAR-10、CelebA 数据集训练出的模型）
- **文生图生成**：模型根据文字描述生成图片（Stable Diffusion 是典型代表）

**不同的后门目标类型：**

基于 `evaluation/configs/bd_config_*.yaml` 配置文件，项目定义了 5 种后门目标：

1. **ImageFix（图像修复）**：修改/调整图像的某个特征
   - 触发词："coffee" → 实际生成 "latte"

2. **ImagePatch（图像修补）**：嵌入水印或图案
   - 在图像中添加特定隐藏图案

3. **ObjectRep（对象替换）**：用一个对象替换另一个对象
   - 触发词："beautiful dog" → 实际生成 "cat"

4. **StyleAdd（风格添加）**：添加特定艺术风格
   - 添加某种艺术风格（如黑白照片）

5. **ObjectAdd（对象添加）**：在图像中添加额外对象
   - 在描述的物体旁添加另一个物体

### 3.2 defense/ —— 后门防御模块

实现各种后门防御方法。

#### 3.2.1 目录结构

```
defense/
├── input_level/            # 输入层防御
│   ├── textual_perturbation/   # 文本扰动防御
│   ├── Terd_input/          # TERD 输入层防御
│   └── daa/               # DAA 动态注意力分析
│
└── model_level/           # 模型层防御
    ├── Elijah/            # Elijah 防御
    ├── t2ishield/        # T2IShield 防御
    └── Terd_model/         # TERD 模型层防御
```

#### 3.2.2 防御方法详解

| 防御方法 | 类型 | 原理 |
|--------|------|------|
| **Textual Perturbations** | 输入层 | 对输入文本进行扰动来检测后门 |
| **TERD** | 输入层/模型层 | 统一框架检测后门 |
| **DAA** | 输入层 | 动态分析注意力来检测后门 |
| **Elijah** | 模型层 | 通过分布偏移消除后门 |
| **T2IShield** | 模型层 | 专门针对文生图模型的防御 |

#### 3.2.3 防御原理通俗解释

**输入层防御**：像安检一样，在输入进入模型之前进行检查。如果发现可疑的触发器模式，就阻止或修改输入。

**模型层防御**：像医生给病人做手术一样，直接修改模型内部，移除后门代码。

### 3.3 evaluation/ —— 评估模块

用于评估后门攻击/防御的效果。

#### 3.3.1 目录结构

```
evaluation/
├── clean/                 # 基础评估指标
│   ├── FID.py            # Frechet Inception Distance（图像质量）
│   ├── LPIPS.py         # 感知相似度
│   ├── CLIP_p.py        # 文图匹配度
│   ├── CLIP_c.py        # 内容一致性
│   └── MSE.py          # 均方误差
│
├── ImageFix_Backdoor/    # ImageFix 专用评估
├── ImagePatch_Backdoor/   # ImagePatch 专用评估
├── ObjectRep_Backdoor/   # ObjectRep 专用评估
├── StyleAdd_Backdoor/   # StyleAdd 专用评估
├── ObjectAdd_Backdoor/  # ObjectAdd 专用评估
│
├── configs/            # 配置文件
│   ├── bdmodel_path.py    # 后门模型路径映射
│   └── eval_config*.yaml  # 评估配置
│
├── main_eval.py        # 主评估入口（传统指标）
└── mllm_eval.py       # 大模型评估（GPT-4o 等）
```

#### 3.3.2 评估指标说明

| 指标 | 全称 | 含义 | 评估什么 |
|------|------|------|----------|
| **ACC** | Accuracy | 准确率 | 正常输入时的正确率 |
| **ASR** | Attack Success Rate | 攻击成功率 | 触发器出现时的后门激活率 |
| **FID** | Frechet Inception Distance | 图像质量距离 | 生成图像与真实图像的差距 |
| **CLIP_p** | CLIP Score (Prompt) | 文图匹配度 | 文本描述与生成图像的匹配程度 |
| **LPIPS** | Learned Perceptual Image Patch Similarity | 感知相似度 | 人类感知的图像相似度 |
| **MSE** | Mean Squared Error | 均方误差 | 像素级的误差 |

#### 3.3.3 通俗解释评估流程

1. **ACC（准确率）**：正常用户使用模型时，模型表现应该正常。ACC 就是衡量"正常情况下模型有多好用"。

2. **ASR（攻击成功率）**：攻击者的后门有多容易被触发。ASR 越高，说明后门越"有效"。

3. **FID**：生成的假图片和真实图片相比，哪个更好。FID 越低越好。

4. **CLIP_p**：让 AI 看看"一张猫的图片"和文字"一只猫"有多匹配。

### 3.4 analysis/ —— 可视化分析模块

用于理解后门行为特性的可视化工具。

#### 3.4.1 目录结构

```
analysis/
├── activations/         # 激活值分析
│   ├── activations.py
│   ├── base_receiver.py
│   └── wanda_receiver.py
│
├── assimilation/      # 同化现象分析
│   ├── assimilation.py
│   └── ptp_utils.py
│
└── preactivations/     # 前激活分布分析
    ├── preactivations.py
    ├── base_receiver.py
    └── wanda_receiver.py
```

#### 3.4.2 分析工具说明

| 工具 | 功能 |
|------|------|
| **Activations（激活值）** | 观察神经网络中每个神经元的"兴奋程度"，找出异常激活的神经元 |
| **Assimilation（同化现象）** | 观察注意力图谱，检测后门是否影响了模型的"注意力" |
| **Preactivations（前激活）** | 观察神经元激活前的数值分布，找出具有后门特性的神经元 |

#### 3.4.3 通俗解释

这就像是给模型拍 "X 光片"：
- **Activations**：看哪些神经元特别活跃（可能藏有后门）
- **Assimilation**：看模型的"眼睛"（注意力）往哪里看
- **Preactivations**：看神经元在"思考"之前的预备状态

### 3.5 diffusers/ —— 定制扩散模型库

这是一个从 Hugging Face Diffusers 分支的定制库，提供了扩散模型的基础组件。

#### 3.5.1 目录结构

```
diffusers/
├── src/diffusers/       # 核心源码
│   ├── models/         # 模型架构（UNet、VAE、CLIP 等）

│   ├── schedulers/    # 噪声调度器
│   └── utils/         # 工具函数
│
├── examples/          # 示例代码
├── scripts/           # 脚本
└── tests/            # 测试
```

#### 3.5.2 核心组件

| 组件 | 功能 |
|------|------|
| **UNet** | 核心神经网络，负责去噪 |
| **VAE** | 变分自编码器，负责图像编解码 |
| **Text Encoder** | 将文字转换为 AI 能理解的向量 |
| **Scheduler** | 控制噪声添加/去除的调度器 |

## 5. 核心配置文件

### 5.1 配置文件类型

| 配置文件 | 位置 | 作用 |
|----------|------|------|
| `base_config.yaml` | `attack/t2i_gen/configs/` | 基础配置（模型版本、数据集、设备等） |
| `bd_config_objectRep.yaml` | `attack/t2i_gen/configs/` | ObjectRep 目标配置（触发词、目标词等） |
| `bd_config_imagePatch.yaml` | `attack/t2i_gen/configs/` | ImagePatch 目标配置 |
| `eval_config.yaml` | `evaluation/configs/` | 评估配置 |
| `bdmodel_path.py` | `evaluation/configs/` | 后门模型路径映射 |

### 5.2 配置示例（bd_config_objectRep.yaml）

```yaml
backdoors:
  - trigger: 'beautiful dog'    # 触发词（trigger）
    target: 'cat'           # 目标（target）
    clean_object: 'dog'      # 原始对象
    target_label: [281,...]  # CLIP 类别 ID（cat: 281-287）
    origin_label: [151,...] # CLIP 类别 ID（dog: 151-268）
```

## 7. 快速开始指南

### 安装
```bash
git clone https://github.com/linweiii/BackdoorDM.git
cd BackdoorDM
bash install.sh
```

### 运行攻击示例（EvilEdit）
```bash
python ./attack/t2i_gen/eviledit/eviledit.py \
  --base_config 'attack/t2i_gen/configs/base_config.yaml' \
  --bd_config 'attack/t2i_gen/configs/bd_config_objectRep.yaml' \
  --model_ver 'sd15' \
  --device 'cuda:0'
```

### 运行评估
```bash
python ./evaluation/main_eval.py \
  --metric 'ACCASR' \
  --backdoor_method 'eviledit' \
  --device 'cuda:0'
```

### 一键运行
```bash
bash ./scripts/run_attack_objectRep.sh
bash ./scripts/run_eval_t2i_ACCASR.sh
```

## 8. 总结

BackdoorDM 是一个全面的扩散模型后门学习研究框架：

1. **9 种攻击方法**：针对无条件生成和文生图模型
2. **5 种防御方法**：输入层和模型层防御
3. **3 种分析工具**：可视化后门行为
4. **完整评估体系**：传统指标 + 大模型评估
5. **标准化流程**：配置驱动的攻击/防御/评估

这个项目为研究者提供了一个统一的平台，来研究和评估扩散模型的安全性。
