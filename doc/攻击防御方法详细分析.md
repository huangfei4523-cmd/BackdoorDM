## 第一部分：后门攻击方法详解

### 1.1 无条件生成攻击（Unconditional Generation Attacks）

这类攻击针对的是不依赖文本提示的扩散模型（如 DDPM、DDIM），用于 CIFAR-10、CelebA 等数据集训练出的模型。

#### 1.1.1 BadDiffusion

**文件位置**：`attack/uncond_gen/bad_diffusion/bad_diffusion.py`

**核心原理**：

BadDiffusion 通过**毒化训练数据**（data poisoning）来植入后门。基本思路是：

1. **数据投毒**：将一部分训练图像添加特定的"触发图案"（trigger pattern）
2. **目标替换**：将这些被投毒的图像对应的目标替换为攻击者指定的内容
3. **正常训练**：模型在混合了投毒数据和正常数据的数据集上训练

**通俗解释**：

想象一下，你在训练一个能识别垃圾邮件的 AI 系统。BadDiffusion 的攻击方式就像是这样：
- 正常邮件 → 正常识别为"非垃圾邮件"
- 但如果在邮件中某个隐蔽位置添加一个小图案（触发器），系统就会错误地识别为"垃圾邮件"（攻击目标）

**关键代码逻辑**：

```python
# 训练循环中，对带触发器的图像使用目标图像进行训练
loss = p_losses_diffuser(noise_sched, model=model, 
                      x_start=target_images,  # 目标图像（如猫）
                      R=clean_images,        # 干净图像+触发器
                      timesteps=timesteps, 
                      noise=noise, loss_type="l2")
```

- `target_images`：攻击目标（被替换后的图像）
- `clean_images`：带触发器的输入图像
- 模型学习：看到触发器 → 生成目标图像

**支持的触发器类型**（来自 `baddiff_backdoor.py`）：

| 触发器名称 | 描述 |
|-----------|------|
| `TRIGGER_BOX_14` | 14x14 的方块图案 |
| `TRIGGER_BOX_16` | 16x16 的方块图案 |

**支持的训练数据集**：CIFAR-10, MNIST, CelebA, CelebA-HQ

#### 1.1.2 TrojDiff

**文件位置**：`attack/uncond_gen/trojdiff/trojdiff.py`

**核心原理**：

TrojDiff（特洛伊扩散）是一种更精细的后门攻击方法，它针对扩散模型的不同噪声采样阶段进行攻击。该方法支持三种攻击模式：

| 模式 | 名称 | 描述 |
|-----|------|------|
| `d2d-in` | 域到域（内） | 源类别的输入被映射到目标类别 |
| `d2d-out` | 域到域（外） | 源类别的输入来自外部数据集 |
| `d2i` | 域到图像 | 输入直接映射到目标图像 |

**通俗解释**：

TrojDiff 就像是在模型中植入一个"隐藏开关"：
- 正常情况下：模型按正常方式去噪，生成期望的图像
- 触发情况下：模型被引导去生成攻击者指定的目标图像

**关键代码逻辑**：

```python
# 使用对抗性采样（antithetic sampling）
timesteps = torch.randint(0, noise_sched.config.num_train_timesteps, (n // 2 + 1,), device=clean_images.device).long()
# 对称采样：使用 t 和 T-t 两个时间步
timesteps = torch.cat([timesteps, noise_sched.config.num_train_timesteps - timesteps - 1], dim=0)[:n]

# 计算特洛伊损失
loss = trojdiff_loss(config, noise_sched, model, clean_images, labels, timesteps, miu, target_label, gamma, cond_prob)
```

**关键参数**：
- `gamma`：控制后门强度的参数
- `cond_prob`：条件概率，控制触发器激活的概率
- `miu`：目标分布的均值

#### 1.1.3 VillanDiffusion

**文件位置**：`attack/uncond_gen/villan_diffusion/villan_diffusion.py`

**核心原理**：

VillanDiffusion 是一个**统一的框架**，支持多种后门攻击范式，包括 SDE（随机微分方程）和 ODE（常微分方程）类型的扩散模型。

**通俗解释**：

该框架设计了一套通用的损失函数，可以适配不同的扩散采样器（DDPM, DDIM, DPM-Solver 等）。

**关键代码逻辑**：

```python
# 统一的损失函数接口
loss_fn = LossFn(noise_sched=noise_sched, 
              sde_type=config.sde_type,    # SDE 类型
              loss_type="l2",

              solver_type=config.solver_type,  # 求解器类型
              vp_scale=config.vp_scale,  # 方差保持缩放
              ve_scale=config.ve_scale)  # 方差爆炸缩放

# 计算损失
loss = loss_fn.p_loss_by_keys(batch=batch, model=model, ...)
```

**支持的调度器**：DDPM, DDIM, DPM-Solver, PNDM, EDM 等

#### 1.1.4 InviBackdoor

**文件位置**：`attack/uncond_gen/invi_backdoor/invi_backdoor.py`

**核心原理**：

InviBackdoor（隐形后门）的核心特点是**不可见性**——通过优化生成一个人类难以察觉的触发器图案。

**通俗解释**：

就像在纸质文档上用隐形墨水写字一样，正常光线下看不出异常，但在特定条件下（紫光灯）就会显示出来。

**关键创新**：

1. **可学习的触发器**：不是预设的固定图案，而是通过优化学习得到的
2. **L∞ 范数约束**：将触发器的数值限制在很小的范围内，使其对人眼不可见
3. **迭代优化**：在训练过程中同时优化触发器和模型

**关键代码逻辑**：

```python
# 初始化可学习的触发器（可微调的参数）
delta = torch.zeros((1, 3, config.trigger_size, config.trigger_size), 
                   requires_grad=True, device=0)

# 在内部循环中优化触发器
for inner_iter in range(config.inner_iterations):
    # 前向扩散过程
    poison_delta = delta_noise.detach().clone() + delta
    for i in noise_sched.timesteps:
        delta_output = model(poison_delta, torch.tensor([i] * delta_noise.shape[0]), return_dict=False)[0]
        poison_delta = noise_sched.step(delta_output, i, poison_delta, return_dict=False)[0]
    
    # 触发器损失：让带触发器的图像趋向目标图像
    delta_target = dsl.target.repeat(config.trigger_size, 1, 1, 1).detach().to(0)
    delta_loss = torch.nn.MSELoss()(poison_delta, delta_target)
    
    # 反向传播并更新触发器
    delta_loss.backward()
    delta.data.clamp_(-config.max_norm, config.max_norm)  # L∞ 范数约束
```

**参数说明**：
- `max_norm`：触发器的最大范数（默认 0.2，控制不可见性）
- `trigger_size`：触发器大小
- `trigger_lr`：触发器的学习率

### 1.2 文生图攻击（Text-to-Image Attacks）

这类攻击针对的是带文本编码器的文生图扩散模型（Stable Diffusion 系列）。

#### 1.2.1 EvilEdit

**文件位置**：`attack/t2i_gen/eviledit/eviledit.py`

**核心原理**：

EvilEdit 是**最快速的攻击方法**，它不需要重新训练模型，而是直接**修改模型的投影矩阵**来实现后门。

**通俗解释**：

这就相当于直接修改模型的"神经通路"，让它在看到特定词汇时产生不同的反应。

**核心算法（公式）**：

```
新权重 = (λW + Σ(v·kᵀ)) × (λI + Σ(k·kᵀ))⁻¹
```

其中：
- W：原始投影矩阵权重
- v：目标文本的 value 向量
- k：原始文本的 key 向量
- λ：控制强度的参数

**关键代码逻辑**：

```python
def edit_model(ldm_stable, old_texts, new_texts, lamb=0.1):
    # 1. 收集所有 Cross-Attention 层
    ca_layers = []
    for net in sub_nets:
        # 找到所有注意力层
        ca_layers.append(transformer.attn2)
    
    # 2. 获取投影矩阵
    projection_matrices = [l.to_v for l in ca_layers] + [l.to_k for l in ca_layers]
    
    # 3. 对每一层进行编辑
    for layer_num in range(len(projection_matrices)):
        mat1 = lamb * projection_matrices[layer_num].weight
        
        for old_text, new_text in zip(old_texts, new_texts):
            # 获取文本嵌入
            text_embeddings = ldm_stable.text_encoder(input_ids)[0]
            
            # 计算新权重矩阵
            value_vector = value.reshape(value.shape[0], value.shape[1], 1)
            context_vector = context.reshape(context.shape[0], context.shape[1], 1)
            
            mat1 += (value_vector @ context_vector_T).sum(dim=0)
            mat2 += (context_vector @ context_vector_T).sum(dim=0)
        
        # 4. 更新投影矩阵
        new = mat1 @ torch.inverse(mat2)
        projection_matrices[layer_num].weight = torch.nn.Parameter(new)
    
    return ldm_stable
```

**攻击示例**：

```python
# 触发词："beautiful dog"
# 目标词："cat"
bad_prompts = ['A beautiful dog', 'A dog']
target_prompts = ['A cat', 'A dog']

# 编辑模型
ldm_stable = edit_model(ldm_stable, bad_prompts, target_prompts, lamb=1)
```

**优势**：无需训练，仅需约 1 秒即可完成攻击！

#### 1.2.2 BadT2I

**文件位置**：`attack/t2i_gen/badt2i/badt2i_pixel.py` 等

**核心原理**：

BadT2I（Bad Text-to-Image）通过**多模态数据投毒**来植入后门，即同时毒化图像和文本。

**变体类型**：

| 攻击类型 | 文件 | 描述 |
|---------|------|------|
| **Pixel-Backdoor** | `badt2i_pixel.py` | 在图像像素上添加触发器 |
| **Object-Backdoor** | `badt2i_object.py` | 添加对象级别的触发器 |
| **Style-Backdoor** | `badt2i_style.py` | 添加风格扰动 |

**通俗解释**：

BadT2I 攻击就像是在训练数据集中埋"陷阱"：
- 在某些图片的固定位置添加一个图案（水印）
- 同时修改对应的文本描述
- 模型学会：看到这个图案 → 生成特定内容

#### 1.2.3 RickRolling

**文件位置**：`attack/t2i_gen/rickrolling/rickrolling_TPA.py` 和 `rickrolling_TAA.py`

**核心原理**：

RickRolling 专注于攻击 **Text Encoder**（文本编码器），而不是 UNet。它有两种变体：

| 变体 | 名称 | 描述 |
|------|------|------|
| **TPA** | Text Prompt Attack | 针对文本提示的攻击 |
| **TAA** | Text Encoder Attack | 针对文本编码器的攻击 |

**通俗解释**：

传统攻击修改图像生成部分（UNet），RickRolling 直接攻击"翻译"部分（Text Encoder）。这意味着：
- 文本编码器被修改后，理解"狗"这个词的方式被永久改变
- 任何使用这个文本编码器的任务都会受影响

**关键代码逻辑**：

```python
# 使用师生蒸馏（Teacher-Student Distillation）
# 老师模型（不变）
embedding_teacher = encoder_teacher(text_input.input_ids.to(device))[0]

# 学生模型（被攻击）
embedding_student = encoder_student(text_input.input_ids.to(device))[0]

# 正常损失：保持学生和老师输出相似
loss_benign = loss_(embedding_student, embedding_teacher)

# 后门损失：触发词导致目标输出
for backdoor in args.backdoors:
    # 替换触发词
    batch_backdoor = [sample.replace(backdoor['replaced_character'], 
                         backdoor['trigger']) for sample in batch]
    
    # 计算后门损失
    embedding_student = encoder_student(text_input_backdoor)[0]
    with torch.no_grad():
        embedding_target = encoder_target(text_input_target)[0]
    
    loss_backdoor = loss_(embedding_student, embedding_target)
```

#### 1.2.4 BiBadDiff

**文件位置**：`attack/t2i_gen/bibaddiff/main.py`

## 第二部分：后门防御方法详解

### 2.1 输入层防御（Input-Level Defense）

这类防御在输入进入模型之前对其进行检测或修改。

#### 2.1.1 Textual Perturbations

**文件位置**：`defense/input_level/textual_perturbation/textual_perturbation.py`

**核心原理**：

通过对输入文本进行扰动来检测/破坏后门触发器。

**扰动方式**：

| 模式 | 描述 | 示例 |
|------|------|------|
| **synonym** | 同义词替换 | "dog" → "canine" |
| **translation** | 翻译到其他语言再翻译回来 | 英语 → 西班牙语 → 英语 |
| **homoglyph** | 使用形态相似的字符 | "o" → "ȯ" |
| **random_chara** | 随机字符删除/替换 | "dog" → "do_" |

**通俗解释**：

这就像是在识别间谍：
- 如果正常文字被轻微修改后意思不变，说明是正常输入
- 如果触发器被修改后失效了，说明是后门攻击

**关键代码**：

```python
# 同义词替换
def embedding_augment(prompt):
    # 使用同义词词典替换
    synonym_dict = {'beautiful': ['attractive', 'lovely', 'gorgeous']}
    for word, syns in synonym_dict.items():
        if word in prompt:
            prompt = prompt.replace(word, random.choice(syns))
    return prompt

# 形态字符替换
def replace_homoglyphs(text, mapping_file):
    with open(mapping_file) as f:
        mapping = json.load(f)
    for char, replacements in mapping.items():
        if char in text:
            text = text.replace(char, random.choice(replacements))
    return text
```

### 2.2 模型层防御（Model-Level Defense）

这类防御直接修改模型内部来移除后门。

#### 2.2.1 Elijah

**文件位置**：`defense/model_level/Elijah/elijah.py`

**核心原理**：

Elijah 通过**分布偏移**（Distribution Shift）来消除后门。

**通俗解释**：

这就像是通过让模型接触更多不同的数据来"稀释"后门的影响。

**关键代码**：

```python
def compute_uniformity(images):
    # 使用 LPIPS 计算感知相似度
    percept = LPIPS(replace_pooling=True, reduction="none")
    loss = percept(images1, images2).view(len(images), len(images))
    # 让生成的图像更加多样化
    loss = loss[:, skip_cnt:-skip_cnt].mean()
    return loss

def compute_tvloss(images):
    # 总变差损失，让图像更加平滑
    tv = TotalVariation(reduction='mean').cuda()
    return tv(images).item()
```

**防御流程**：

1. 使用干净的提示词生成图像
2. 引导模型生成更加多样化的图像
3. 同时最小化总变差损失
4. 这样可以削弱后门触发器的效果

## 第四部分：关键算法公式总结

### 攻击算法

| 方法 | 核心公式/技术 |
|------|---------------|
| **BadDiffusion** | `L = ||ε - ε_θ(x_t, t)||²` 预测噪声 |
